# RayText — Nim 中文文本渲染模块

基于 [raylib](https://www.raylib.com/) 和 [naylib](https://github.com/planetis-m/naylib)，支持中英文混排、CJK 统一汉字、Nerd Font 图标、自动换行、按需动态加载字形。

## 安装

```bash
nimble install naylib
```

将 `raytext.nim` 放入项目目录即可使用。

## 快速入门

```nim
import raytext

let 字体 = 加载字体全("font.ttf", 48)

initWindow(800, 600, "RayText")
while not windowShouldClose():
  beginDrawing()
  clearBackground(RayWhite)
  绘制文本(字体, "你好世界！Hello World!",
            Vector2(x: 20, y: 20), 32, 2, DarkBlue)
  绘制文本换行(字体, "长文本自动换行测试...",
                Rectangle(x: 20, y: 80, width: 300, height: 200),
                20, 2, 4, DarkBlue)
  endDrawing()
```

## 两种加载模式

### 方式一：静态预加载

启动时一次性加载全部字形，适合已知文本内容的应用。

```nim
let f = 加载字体("font.ttf")                         # 常用子集（≈29K）
let f = 加载字体("font.ttf", 子集 = 汉字全集)          # 仅汉字（≈94K）
let f = 加载字体全("font.ttf")                        # 所有码点（≈127K）
```

| 函数 | 说明 |
|------|------|
| `加载字体(路径, 字号, 子集, 额外码点)` | 按码点范围加载 |
| `加载字体全(路径, 字号)` | 加载所有支持的码点（≈127K） |
| `加载字体从内存(类型, 数据, 字号, 子集, 额外码点)` | 从内存缓冲区加载 |

### 方式二：动态按需加载

启动只加载基础字符（≈50ms），遇到新字符时自动追加。适合文本输入、编辑器等场景。

```nim
var df = 初始化动态字体("font.ttf", 32)
预加载文本(df, ["你好世界", "→ ■ ★"])   # 批量预加载
绘制文本(df, "你好世界", Vector2(x: 10, y: 10), 24, 2, Black)
```

| 函数 | 说明 |
|------|------|
| `初始化动态字体(路径, 字号)` | 创建动态字体，初始≈50ms |
| `确保已加载(df, 文本)` | 确保文本字符已加载 |
| `预加载码点范围(df, 子集)` | 批量加载码点范围 |
| `预加载文本(df, 文本集)` | 批量加载文本中的字符 |
| `已加载数(df)` | 当前已加载字符数 |

## 码点范围枚举（18 个）

所有连续区块已合并，相邻的小范围整合为单一枚举成员。

### 拉丁 / 符号

| 枚举 | 范围 | 内容 |
|------|------|------|
| `r基本拉丁` | 0x0020–0x007E | ASCII 可打印字符 |
| `r拉丁补充` | 0x00A0–0x00FF | · × ÷ |
| `r常用标点` | 0x2000–0x206F | — … ‐ – |
| `r箭头` | 0x2190–0x21FF | → ← ↑ ↓ ⇒ |
| `r数学运算符` | 0x2200–0x22FF | ≈ ≠ ≤ ≥ ∈ √ ∞ ± |
| `r图形符号` | **0x25A0–0x27BF** | ■□▲△●○★☆♥♠➜（合并：几何 + 杂项 + 丁巴特） |

### 汉字辅助（合并）

| 枚举 | 范围 | 合并前 |
|------|------|--------|
| `r汉字辅助` | **0x2E80–0x312F** | 部首拓展 + 康熙部首 + 结构描述 + CJK 标点 + 日文假名 + 片假名 + 注音符号 |
| `r注音笔画` | **0x31A0–0x31EF** | 注音拓展 + 汉字笔画 |

### CJK 核心

| 枚举 | 范围 | 说明 |
|------|------|------|
| `rCJK扩展A` | 0x3400–0x4DBF | 㐀–䶿 (6,592 字) |
| `r基本汉字` | 0x4E00–0x9FFF | 一–鿿 (20,992 字) |

### CJK 兼容与变体

| 枚举 | 范围 | 说明 |
|------|------|------|
| `r兼容汉字` | 0xF900–0xFAFF | 豈–﫿 (512 字) |
| `r汉字变体` | **0xFE10–0xFE6F** | 垂直形式 + 兼容形式 + 小型变体（合并） |
| `r全角半角` | 0xFF00–0xFFEF | 全角字母数字、括号 |
| `r象形符号标点` | 0x16FE0–0x16FFF | 𖿠–𖿿 |

### CJK 扩展（超大）

| 枚举 | 范围 | 说明 |
|------|------|------|
| `r扩展BtoF` | **0x20000–0x2EBE0** | 扩展 B~F 合并（约 60K 字） |
| `r兼容汉字扩展` | 0x2F800–0x2FA1F | (544 字) |
| `r扩展GtoH` | **0x30000–0x323AF** | 扩展 G+H 合并（约 9K 字） |

### Nerd Font 图标

| 枚举 | 范围 | 说明 |
|------|------|------|
| `rNerd字体` | **0xE000–0xF8FF** | 全部 NF 图标合并（Powerline / Devicons / FontAwesome / Material） |

## 预定义子集常量

| 常量 | 码点数 | 包含 |
|------|--------|------|
| `拉丁符号集` | ≈500 | 拉丁 + 标点 + 箭头 + 数学 + 图形 |
| `汉字核心` | ≈29K | 汉字 + 扩展A + 辅助 + 变体 + 全角 |
| `汉字扩展` | ≈70K | 扩展 B~H + 兼容扩展（超大，按需） |
| `NF字体集` | ≈6K | Nerd Font 图标 |
| `常用子集` | ≈29K | 拉丁符号集 + 汉字核心 |
| `汉字全集` | ≈94K | 汉字核心 + 汉字扩展 |
| `完整子集` | ≈127K | 所有范围（含 NF） |

## 自定义子集

```nim
# 只加载特定范围
let f = 加载字体("font.ttf", 子集 = {r基本拉丁, r基本汉字})

# 组合已有子集
let f = 加载字体("font.ttf", 子集 = 常用子集 + NF字体集)

# 动态字体预加载码点范围
预加载码点范围(字体, {r图形符号, r数学运算符})
```

## 常见问题

### 启动卡顿？

预加载 20,000+ 字形需要一定时间。改用动态字体：

```nim
var df = 初始化动态字体("font.ttf", 32)  # ≈50ms
预加载文本(df, ["你的界面文本", "..."])      # 批量加载
```

### 中文显示问号？

确保使用支持中文的字体（Maple Mono CN、Noto Sans CJK 等），
且用 `绘制文本(字体, ...)` 而非 raylib 默认 `drawText`。

### Nerd Font 不显示？

使用 NF 变体（如 `MapleMono-NF-CN.ttf`），加载 NF 码点：

```nim
let f = 加载字体("MapleMono-NF-CN.ttf", 子集 = 常用子集 + NF字体集)
```

### 输入法退格误删？

raylib 的已知局限——IME 退格和键盘退格事件无法区分。
不影响功能，只是退格可能多删除一个已提交字符。
